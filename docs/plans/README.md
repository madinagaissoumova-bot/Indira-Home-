# Plans De Travail

Ce dossier sert a organiser le travail avant de modifier le code.

Regle simple :

1. creer un plan dans `docs/plans/en-cours/` ;
2. executer le plan point par point ;
3. verifier le resultat ;
4. deplacer le plan dans `docs/plans/termines/` ;
5. pousser sur GitHub seulement quand le plan est termine.

Un seul plan en cours doit exister a la fois, sauf demande explicite.
