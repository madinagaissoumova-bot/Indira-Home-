# Plans De Travail

Ce dossier sert uniquement aux petits plans de travail quotidiens.

Il ne remplace pas :

- `docs/roadmap/` : la direction generale du projet ;
- `docs/development-plans/` : les grands lots, tickets et statuts de la V1.

Organisation :

- `docs/plans/en-cours/` : le plan que l'on execute maintenant ;
- `docs/plans/termines/` : les plans deja finis et verifies.

Regle simple :

1. creer un plan dans `docs/plans/en-cours/` ;
2. travailler selon ce plan ;
3. verifier ;
4. deplacer le plan dans `docs/plans/termines/` ;
5. pousser sur GitHub seulement quand le plan est termine.
