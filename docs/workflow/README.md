# Workflow De Travail

Ce dossier decrit seulement notre methode de travail quotidienne.

Il ne remplace pas :

- `docs/roadmap/` : direction generale du projet ;
- `docs/development-plans/` : grands lots, tickets, statut V1, securite et production.

Regle simple :

1. avant un changement significatif, remplir `docs/workflow/plan-en-cours.md` ;
2. executer le travail ;
3. verifier ;
4. vider ou remettre a zero `docs/workflow/plan-en-cours.md` ;
5. pousser sur GitHub seulement quand le plan courant est termine.
