# Critères d'Acceptation

Chaque exigence importante doit avoir des critères testables.

## Format

```gherkin
Given [contexte]
When [action]
Then [résultat observable]
```

## Critères

### FR-001

```gherkin
Given
When
Then
```

## Definition of Ready

Une fonctionnalité est prête à développer quand :

- Le besoin utilisateur est clair.
- Les critères d'acceptation sont testables.
- Les cas d'erreur principaux sont documentés.
- Les dépendances externes sont identifiées.

## Definition of Done

Une fonctionnalité est terminée quand :

- Les critères d'acceptation sont satisfaits.
- Les tests liés passent.
- La documentation nécessaire est mise à jour.
- Les décisions techniques importantes sont tracées.
