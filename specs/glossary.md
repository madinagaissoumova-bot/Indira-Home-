# Glossaire

| Terme | Définition |
| --- | --- |
| Specification-Driven Development | Approche où les specifications guident l'implémentation et les tests. |
| Exigence fonctionnelle | Comportement que le système doit fournir. |
| Exigence non fonctionnelle | Qualité attendue du système, comme performance, sécurité ou accessibilité. |
| Critère d'acceptation | Condition vérifiable qui permet de dire qu'une exigence est satisfaite. |
| ADR | Architecture Decision Record, document court qui trace une décision technique importante. |
